# Потом что-то серьезное напишу

## Вводные:

Есть сервис, который пользователь хочет протестировать с yandex-tank.
Но пользователь не хочет вручную прописывать запросы для каждого эндпоинта для каждого метода (особенно если и тех и
других много разных).
У него есть, или он получает из своего кода, файл с документацией по своему сервису в формате OpenApi.
Запускает вот это приложение для генерации и отправляет к нему запрос в теле которого лежат и файл openApi, и желаемые
значения для будущего конфига (вроде адреса хоста).
Приложение ему выплевывает архив в котором лежат файлы конфигурации.
На этом, по идее, жизненный цикл приложения заканчивается и пользователь уже дальше сам либо правит конфиги как ему
нравится, либо сразу запускает танк.

---

Код весь на Spring фреймворке написан, очень хотел начать изучать (читал спешно, но вроде косяки еще прячутся).
В ближайшее время добавлю страницу с полями и кнопками, чтоб живые человеки пользоваться могли (пока что все через
Postman проверяется).

---

Чтоб протестить написал отдельно сервис с парой методов, но чуть увлекся и БД прикрутилась.
В целом он занял бы час максимум, если бы в docker его не пришлось паковать (было больно и грустно...), а паковать
пришлось исключительно из-за БД...

Этот тестовый сервис чуть позже тоже докину сюда (отдельной репой наверно).

---

Еще момент: на гитхабе танка лежит ссылка на сайт overload (от них же), где графики получившиеся посмотреть можно.
НО есть нюанс. Либо у меня телефон с ноутбуком очень не любят яндекс, либо этого сайта больше не существует.
Так-то это проблемы пользователя (да и есть вроде другие способы графики посмотреть), но просто забавно, что офф-сайт
выбрал быть счастливым.
